### 시뮬레이션 모델
캔들의 `[high, low]`로 그리드 라인 cross 측정:
- 라인이 `low ≤ line ≤ high` 안에 있으면 cross 1회
- `open` / `close` 위치로 up-cross / down-cross 분류
- 양쪽 wick만 닿은 경우(같은 쪽으로 닫힘) up/down 0.5씩 (보수적)

라운드트립 = `min(up_count, down_count)` per line. 1 RT 수익:
```
spacing × qty − (line[i] × qty + line[i+1] × qty) × 0.001
```
수수료 0.1% per leg = 0.2% / RT, 총 투자금 $10,000 균등 분할.

### 이탈(OOR) 추적
캔들 close가 `[lower, upper]` 밖이면 그 캔들 duration을 OOR에 가산. 점수 계산 시 OOR 24시간당 -0.5%p 페널티.

### 파라미터 그리드

- **upper**: 2400, 2450, 2500, 2550, 2600, 2700
- **gridNum**: 10, 15, 20, 30
- **mode**: arithmetic, geometric
- 제약: `200 ≤ upper-lower ≤ 800`, 현재가가 `[lower, upper]` 안에 위치

## 핵심 발견

1. **lower=2000이 모든 시간 단위에서 압승.**
   - 4H: ETH 50일 중 8.3일만 $2,000 아래 → OOR 200h
   - lower를 2050으로 올리면 OOR 316h (+58%), 2100이면 472h (+136%)
   - "$2,000 아래 갔던 짧은 기간 손실" < "그리드 안 활동량 이득"

2. **그리드 10개가 거의 항상 최적.**
   - 15/20/30개는 spacing 좁아져 한 RT당 수수료 비중 ↑
   - Top 14가 모두 N=10

3. **arithmetic ≈ geometric (1-2%p 차이).**
   - arithmetic이 spacing 균등해서 직관적 추천

4. **upper sweet spot은 2500-2600.**
   - 2400은 OOR 약간 ↑, 2700까지는 spacing 너무 커서 RT 빈도 ↓

5. **사용자 제안 2100-2400은 차선 옵션 대비도 -1%p.**
   - 하단봇과 분리되어 있다는 장점은 있으나, lower=2100의 OOR 페널티가 큼

6. **내 처음 추천 2250-2500은 백테스트 검증에서 무너짐.**
   - "현재가 중앙형이 안전"이라는 직관이 ETH 최근 환경에 안 맞았음
   - OOR 33일(50일 중 67%) — 사실상 그리드가 거의 안 도는 구성

## 한계

1. **캔들 high/low 기반은 보수적 추정** — 1봉 안의 가격 변동 순서를 모르므로 실제 RT는 더 많을 수 있음
2. **수수료 0.1% per leg 가정** — VIP 등급/리베이트는 미반영
3. **이탈 시 재진입 시점은 모델링 안 함** — 실제로는 이탈 후 재진입까지 기회비용 추가
4. **표본 크기**:
   - 1H 12일 — 신뢰도 낮음
   - 4H 50일 — **메인** (충분)
   - 1D 300일 — long-term이라 좁은 범위 평가 부적합
5. **Slippage / 매수-매도 호가 차이 미반영**
