### 진단 결론 (요약)

OKX UI는 "manual stop"으로 표기 → 사용자 수동 정지 또는 **외부 시스템에서 cancel API 호출**. 4가지 가설 검증 후 **`main_agent.py`의 STOP 분기가 호출됐다**가 결론. 5가지 근본 원인:

1. `score >= 90 → STOP` 백도어 (코드 레벨 강제 종료 경로)
2. 4-trader 패널의 STOP 편향 (risk weight 2x)
3. STOP 가드 부재 (단일 LLM 응답으로 $48k 봇 종료 가능)
4. LLM 프롬프트가 "거래 트레이더" 마인드 → 그리드 봇 운영가가 아님
5. 80%/20% 트리거가 횡보장에서 상시 발화

→ 4/27 작업으로 전부 패치.

## 6. 시스템 변경의 의미

이 3일은 **봇 PnL 측면에선 0원, 시스템 측면에선 가장 큰 리셋**. 핵심 변화 3가지:

1. **STOP 가드 추가** — 단일 LLM 응답으로 봇 정지 불가. 만장일치 + confidence ≥ 9 + 코드 다운그레이드 fallback.
2. **MAINTAIN 기본값 강제** — 동률/모호한 상황은 항상 MAINTAIN으로 기울도록 tie-break 변경.
3. **LLM 마인드셋 교체** — "단기 트레이더"가 아닌 "그리드 봇 오퍼레이터" 페르소나. 횡보장에 STOP 안 누름.

→ 4/26 같은 사고 패턴은 구조적으로 재발 불가. 검증은 후행 운영 데이터 누적으로.
