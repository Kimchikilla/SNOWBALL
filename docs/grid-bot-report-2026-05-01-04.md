## 4. 왜 첫 회전이 손실로 끝났나 — 진단

### 가설: "그리드 하단부 매수가 평단을 밀어올림"

- 4/28 첫 진입 매수 평단: **$2,299.17** (그리드 중앙)
- 4/30 저가 $2,220 도달 시 $2,250 라인 추가 매수 (가설)
- 5/1 회복 시 $2,300 라인 매도가 트리거되면 → **매도가 < 평균 원가**

### 그리드 봇 회계의 함정

OKX 그리드 봇은 **"라인별 매수 → 같은 라인+1 매도" = 그리드 수익**으로 계산하지만, 우리 코드의 `realized_pnl`은 **FIFO 평균 원가 방식**으로 추적. 두 방식의 결과가 단기적으로 다를 수 있음:

| 회계 방식 | 5/1 매도 1 ETH @$2,300의 손익 |
|---|---|
| OKX gridProfit (라인 단위) | $2,300 - $2,250 = **+$50 (수익)** |
| 우리 realized_pnl (FIFO 평균) | $2,300 - $2,299 = **+$1 (거의 0)** |

→ OKX UI에서 보면 봇이 **수익**처럼 보이지만, 우리 봇 상태 파일의 `realized_pnl`엔 평균 진입가 기준이라 거의 0. 추가로 fee $9.49가 차감되면 **순손실**로 기록.

> 즉 **회계 방식 차이로 인한 -$106 보고**일 가능성이 큼. OKX UI의 gridProfit은 +$? 양수일 수 있으니 **반드시 OKX UI에서 별도 확인 필요**.
