## 4. 발견: 그리드 봇 회계의 함정

### 5/10-11 매도가 realized PnL을 거의 안 움직인 이유

매도가 ($2,350) ≈ 평균 cost basis ($2,350.2). 두 가지로 해석 가능:

**OKX gridProfit 회계 (라인별 매칭)**
```
이전 매수가: $2,300 라인
이번 매도가: $2,350 라인
→ 그리드 마진: +$50 × 1.706 ETH = +$85 (수익)
```

**우리 코드 realized_pnl (FIFO 평균원가)**
```
평균 cost basis: $2,350.2
매도가: $2,350.0
→ 마진: -$0.2 × 1.706 ETH = -$0.34 (거의 0)
+ fee -$3.21 → 합 -$3.55
```

→ **OKX UI는 +$85, 우리 코드는 -$3**. **회계 방식 차이만 ~$88**.

### 시사점

| 항목 | 함의 |
|---|---|
| OKX UI gridProfit 기준 누적 | 추정 +$160~$200 (RT 2회 × $50 + 부분 매도 +$85) |
| 우리 코드 realized 기준 누적 | -$80.65 |
| **격차** | **~$240** |

→ **OKX UI에서 직접 봇 gridProfit 확인 필수**. 우리 코드의 realized_pnl은 보수적 회계라 단기적으로 손실로 보일 수 있지만, **거래소 회계로는 양수일 가능성 높음**.

이건 백테스트 ↔ 실측 갭의 원인 중 하나이기도 함 — 백테스트는 OKX 방식 라인별 매칭을 가정했는데 실측 코드는 FIFO. 회계 일치화는 후속 작업으로.
